# 📘 README – Analyse de Code | DevStart Agency
## 🧩 Contexte du projet
Dans le cadre du module « Le code que tu n’as pas écrit », notre groupe a reçu le code source d’une mini-application web développée par un stagiaire.
Ce code :
- fonctionne partiellement
- ne contient pas de documentation
- présente des problèmes de lisibilité et d’organisation<br><br>Notre mission est de comprendre, analyser et documenter ce code sans le modifier.

## ⚙️ Démarche suivie
Notre travail s’est déroulé en trois phases principales :
### 🔹 Phase 1 : Exploration
**Objectif** : Comprendre le code<br>
- **Actions réalisées :**
```
- Lecture complète du code sans modification
- Identification des différentes parties
- Annotation du code avec nos propres mots
- Identification des variables et de leur rôle supposé
```
<br> 👉 À cette étape, nous avons cherché à répondre à :
<br> ***“Que fait ce code ?”***
### 🔹 Phase 2 : Analyse critique
**Objectif** : Evaluer la qualité du code<br>
- **Actions réalisées :**
```
- Identification des problèmes de lisibilité (noms, structure)
- Détection des parties confuses ou mal organisées
- Repérage des blocs de code trop complexes
- Proposition de noms plus explicites pour les variables
```
<br> 👉 À cette étape, nous avons cherché à répondre à :
<br> ***“Ce code est-il bien écrit et compréhensible ?”***
### 🔹 Phase 3 : Documentation
**Objectif** : Expliquer clairement le code
- **Actions réalisées :**
```
- Rédaction du rapport d’analyse
- Organisation des explications
- Préparation de la présentation orale
```
<br> 👉 À cette étape, nous avons cherché à répondre à :
<br> ***“Comment expliquer ce code à quelqu’un d’autre ?”***
## 🔍 Type d’analyse réalisée
Nous avons effectué une :
#### ✅ Analyse globale
- Rôle général de l’application
- Organisation en sections
#### ✅ Analyse détaillée
- Explication fonction par fonction / section par section
- Description du comportement du code
#### ✅ Analyse critique
- Problèmes de lisibilité
- Problèmes de logique
- Problèmes d’organisation
